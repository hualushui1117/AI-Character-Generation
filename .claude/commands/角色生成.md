
## 阶段一：图像生成

### 入口识别

判断用户的初始信息，进入对应入口：

**入口 A — 用户上传了图片**
```
询问："你想直接使用这张图，还是以它为参考生成新图？"
├── 作为参考 → 进入入口 B（上传图 + 描述）
└── 直接使用
     └── 软提示："最佳格式为全身正面图 / 绿幕 / 9:16，你的图是否符合？"
           ├── 符合 → 跳过图像阶段，直接进入【角色命名】
           └── 不符合
                └── 执行图像命令（I2I），以原图为参考转换格式
                     └── 展示结果，进入【满意度循环】
```

**入口 B — 参考图 + 文字描述**
```
收集参考图路径 + 用户文字描述
└── 进入【引导追问机制】
     └── 执行图像命令（I2I），生成 2 张
          └── 进入【满意度循环】
```

**入口 C — 纯文字描述**
```
收集用户文字描述
└── 进入【引导追问机制】
     └── 执行图像命令（T2I），生成 2 张
          └── 进入【满意度循环】
```

---

### 引导追问机制

**5 个角色维度**（按入口 B/C 使用）：

| 维度 | 说明 | 示例 |
|------|------|------|
| 风格 | 整体画风 | 写实 / 动漫 / 3D / 像素 |
| 人设基础 | 性别、年龄感、种族 | 青年女性、精灵族 |
| 外貌特征 | 发色发型、眼睛、肤色 | 银白短发、红色眼睛 |
| 服装装扮 | 职业/风格/材质 | 黑色皮质战斗服、奇幻皮甲 |
| 气质表情 | 性格倾向与情绪基调 | 冷酷、活泼、神秘 |

**规则**：
- 用户未提供的维度由模型合理补全，不强制追问
- 每轮将所有缺失维度一次性批量问完（不分多轮追问）
- 最多追问 3 轮，超出后以现有信息直接生成
- 不向用户展示最终 prompt，直接执行生成

**示例**：
```
用户: "一个冷酷的女刺客"

已覆盖: 性别(女)、气质(冷酷)、职业暗示(刺客)
缺失:   风格、外貌、服装细节

Claude 回复:
还需要几个细节：
1. 画风偏写实还是动漫？
2. 发色和发型有想法吗？（如黑色短发 / 银白长发）
3. 服装风格？（现代战术装 / 奇幻皮甲 / 忍者风）
```

---

### 背景确认

引导追问完成后，生成前询问：

```
"需要为角色生成背景吗？（不需要的话默认绿幕背景，方便后期一键去除）"

├── 否（默认）→ 保留 prompt 中的"纯绿色（#00FF00）绿幕背景" → 进入图像生成
└── 是 → 移除 prompt 中的"纯绿色（#00FF00）绿幕背景"
           软提示："带背景的图片将无法使用最后的透明背景处理"
           可追问用户描述背景（可选）→ 进入图像生成
```

同时将内部状态 `绿幕背景` 设为 `true`（否）或 `false`（是），供透明背景处理阶段判断，并赋值 bash 变量：

```bash
GREEN_SCREEN=true   # 用户选择绿幕背景（否，默认）
# 或
GREEN_SCREEN=false  # 用户选择生成背景（是）
```

### 图像生成 CLI 命令

**T2I（纯文字，入口C）**：
```bash
# 分析用户完整描述，找出视觉上存在合理分歧的维度，拟定两种诠释分别生成
RESULT1=$(pixverse create image \
  --prompt "全身正面图，[纯绿色（#00FF00）绿幕背景，][用户角色描述·诠释A]" \
  --model gemini-3.1-flash \
  --quality 1080p \
  --aspect-ratio 9:16 \
  --count 1 \
  --no-wait \
  --json)

RESULT2=$(pixverse create image \
  --prompt "全身正面图，[纯绿色（#00FF00）绿幕背景，][用户角色描述·诠释B]" \
  --model gemini-3.1-flash \
  --quality 1080p \
  --aspect-ratio 9:16 \
  --count 1 \
  --no-wait \
  --json)
```

**I2I（参考图，入口A不符合/入口B）**：
```bash
RESULT1=$(pixverse create image \
  --prompt "全身正面图，[纯绿色（#00FF00）绿幕背景，][用户角色描述·诠释A]" \
  --image [参考图路径] \
  --model gemini-3.1-flash \
  --quality 1080p \
  --aspect-ratio 9:16 \
  --count 1 \
  --no-wait \
  --json)

RESULT2=$(pixverse create image \
  --prompt "全身正面图，[纯绿色（#00FF00）绿幕背景，][用户角色描述·诠释B]" \
  --image [参考图路径] \
  --model gemini-3.1-flash \
  --quality 1080p \
  --aspect-ratio 9:16 \
  --count 1 \
  --no-wait \
  --json)
```

> `[纯绿色（#00FF00）绿幕背景，]` 根据背景确认步骤的选择决定是否保留。

**解析输出**：
```bash
IMG1_ID=$(echo "$RESULT1" | jq -r '.image_ids[0]')
IMG2_ID=$(echo "$RESULT2" | jq -r '.image_ids[0]')
```

**等待完成**：
```bash
pixverse task wait $IMG1_ID --type image --json
pixverse task wait $IMG2_ID --type image --json
```

获取预览 URL，展示给用户选择（仅展示，不下载）：
```bash
pixverse asset info $IMG1_ID --type image --json
pixverse asset info $IMG2_ID --type image --json
```

### 满意度循环

每次展示 2 张图供用户选择，最多重生 3 次（内部状态图像重生次数）。展示时附上一行对比说明，告知两张图在哪个维度上存在差异，例如：
> 「图一偏冷酷内敛，图二偏强势凌厉——两种气质诠释，选你更想要的方向。」

```
展示 2 张图（展示 asset info 中的 image_url，附差异说明）
 │
 ├── 用户选中一张 → 进入【角色命名】→ 命名完成后下载参考图（见下）
 │
 ├── 局部不满意（有具体反馈）
 │    └── 定向修改对应维度 → 重新生成（图像重生次数 +1）
 │
 ├── 整体不满意
 │    └── 重新引导追问 → 重新生成（图像重生次数 +1）
 │
 └── 已达 3 次上限
      └── 提示："要不要重新描述角色，或者上传一张参考图？"
           ├── 是 → 重置重生次数，回到入口 B/C
           └── 否 → 从现有图中强制选一张继续
```

### 角色命名

```
Claude 提示："请为这个角色取个名字～"
用户输入角色名（2-20字符，如：冰川战士、月光精灵）
└── 确认后，将角色名记入内部状态
```

```bash
CHAR_NAME="[用户输入的角色名]"
```

### 最终参考图下载

角色命名完成后，清空并下载最终参考图：
```bash
# CHOSEN_IMG_ID = 用户选中的那张图的 image_id（IMG1_ID 或 IMG2_ID）
rm -f "output/$CHAR_NAME/image/"*
mkdir -p "output/$CHAR_NAME/image"
TMP_IMG=$(mktemp -d)
pixverse asset download "$CHOSEN_IMG_ID" --type image --dest "$TMP_IMG" --json
mv "$TMP_IMG"/*.* "output/$CHAR_NAME/image/character.png"
rm -rf "$TMP_IMG"
```

写入断点状态：
```bash
cat > .pixverse_state.json << EOF
{
  "phase": "image_done",
  "character_name": "$CHAR_NAME",
  "green_screen": $GREEN_SCREEN,
  "image_path": "output/$CHAR_NAME/image/character.png"
}
EOF
```

→ 进入阶段二（视频生成）

## 阶段二：视频生成

### 执行说明

- 系统**自动执行**，不向用户询问任何选择
- 参考图路径为阶段一下载的文件：`output/[角色名]/image/` 下的图片文件
- 先用 `--no-wait` 并行提交 12 个任务，再统一等待，最后下载

### 4 个状态的提示词

提示词从 `prompt.md` 读取，每个状态3条变体，固定前缀为：
`固定镜头，保持人物形象一致性，背景不变，保持竖屏构图：`

每次生成时将3条变体分别分配给视频1/2/3，确保每个视频动作各有差异。

### 视频生成 CLI 命令

执行前先读取 `prompt.md`，将各状态变体文本赋值到以下变量（固定前缀已含在 prompt.md 的每条完整文本中）：

```bash
# 从 prompt.md 读取各状态3条变体，赋值给以下变量
# （执行前 Claude 须已读取 prompt.md，将对应文本填入）
IDLE_P1="[prompt.md Idle 变体1完整文本]"
IDLE_P2="[prompt.md Idle 变体2完整文本]"
IDLE_P3="[prompt.md Idle 变体3完整文本]"

TALK_P1="[prompt.md Talk 变体1完整文本]"
TALK_P2="[prompt.md Talk 变体2完整文本]"
TALK_P3="[prompt.md Talk 变体3完整文本]"

THINK_P1="[prompt.md Think 变体1完整文本]"
THINK_P2="[prompt.md Think 变体2完整文本]"
THINK_P3="[prompt.md Think 变体3完整文本]"

LISTEN_P1="[prompt.md Listen 变体1完整文本]"
LISTEN_P2="[prompt.md Listen 变体2完整文本]"
LISTEN_P3="[prompt.md Listen 变体3完整文本]"
```

```bash
# 参考图路径（阶段一已下载的图片）
CHAR_IMG="output/$CHAR_NAME/image/character.png"

# 1. 并行提交 12 个任务（--no-wait 立即返回 video_id）

# --- idle ---
IDLE1_RESULT=$(pixverse create transition \
  --images "$CHAR_IMG" "$CHAR_IMG" \
  --prompt "$IDLE_P1" \
  --quality 1080p --duration 3 --no-wait --json)
IDLE1_ID=$(echo "$IDLE1_RESULT" | jq -r '.video_id')

IDLE2_RESULT=$(pixverse create transition \
  --images "$CHAR_IMG" "$CHAR_IMG" \
  --prompt "$IDLE_P2" \
  --quality 1080p --duration 3 --no-wait --json)
IDLE2_ID=$(echo "$IDLE2_RESULT" | jq -r '.video_id')

IDLE3_RESULT=$(pixverse create transition \
  --images "$CHAR_IMG" "$CHAR_IMG" \
  --prompt "$IDLE_P3" \
  --quality 1080p --duration 3 --no-wait --json)
IDLE3_ID=$(echo "$IDLE3_RESULT" | jq -r '.video_id')

# --- talk ---
TALK1_RESULT=$(pixverse create transition \
  --images "$CHAR_IMG" "$CHAR_IMG" \
  --prompt "$TALK_P1" \
  --quality 1080p --duration 3 --no-wait --json)
TALK1_ID=$(echo "$TALK1_RESULT" | jq -r '.video_id')

TALK2_RESULT=$(pixverse create transition \
  --images "$CHAR_IMG" "$CHAR_IMG" \
  --prompt "$TALK_P2" \
  --quality 1080p --duration 3 --no-wait --json)
TALK2_ID=$(echo "$TALK2_RESULT" | jq -r '.video_id')

TALK3_RESULT=$(pixverse create transition \
  --images "$CHAR_IMG" "$CHAR_IMG" \
  --prompt "$TALK_P3" \
  --quality 1080p --duration 3 --no-wait --json)
TALK3_ID=$(echo "$TALK3_RESULT" | jq -r '.video_id')

# --- think ---
THINK1_RESULT=$(pixverse create transition \
  --images "$CHAR_IMG" "$CHAR_IMG" \
  --prompt "$THINK_P1" \
  --quality 1080p --duration 3 --no-wait --json)
THINK1_ID=$(echo "$THINK1_RESULT" | jq -r '.video_id')

THINK2_RESULT=$(pixverse create transition \
  --images "$CHAR_IMG" "$CHAR_IMG" \
  --prompt "$THINK_P2" \
  --quality 1080p --duration 3 --no-wait --json)
THINK2_ID=$(echo "$THINK2_RESULT" | jq -r '.video_id')

THINK3_RESULT=$(pixverse create transition \
  --images "$CHAR_IMG" "$CHAR_IMG" \
  --prompt "$THINK_P3" \
  --quality 1080p --duration 3 --no-wait --json)
THINK3_ID=$(echo "$THINK3_RESULT" | jq -r '.video_id')

# --- listen ---
LISTEN1_RESULT=$(pixverse create transition \
  --images "$CHAR_IMG" "$CHAR_IMG" \
  --prompt "$LISTEN_P1" \
  --quality 1080p --duration 3 --no-wait --json)
LISTEN1_ID=$(echo "$LISTEN1_RESULT" | jq -r '.video_id')

LISTEN2_RESULT=$(pixverse create transition \
  --images "$CHAR_IMG" "$CHAR_IMG" \
  --prompt "$LISTEN_P2" \
  --quality 1080p --duration 3 --no-wait --json)
LISTEN2_ID=$(echo "$LISTEN2_RESULT" | jq -r '.video_id')

LISTEN3_RESULT=$(pixverse create transition \
  --images "$CHAR_IMG" "$CHAR_IMG" \
  --prompt "$LISTEN_P3" \
  --quality 1080p --duration 3 --no-wait --json)
LISTEN3_ID=$(echo "$LISTEN3_RESULT" | jq -r '.video_id')

# 写入断点状态（12 个 ID 均已拿到，万一后续中断可直接恢复等待）
cat > .pixverse_state.json << EOF
{
  "phase": "videos_submitted",
  "character_name": "$CHAR_NAME",
  "green_screen": $GREEN_SCREEN,
  "image_path": "output/$CHAR_NAME/image/character.png",
  "video_ids": {
    "idle1": "$IDLE1_ID", "idle2": "$IDLE2_ID", "idle3": "$IDLE3_ID",
    "talk1": "$TALK1_ID", "talk2": "$TALK2_ID", "talk3": "$TALK3_ID",
    "think1": "$THINK1_ID", "think2": "$THINK2_ID", "think3": "$THINK3_ID",
    "listen1": "$LISTEN1_ID", "listen2": "$LISTEN2_ID", "listen3": "$LISTEN3_ID"
  }
}
EOF

# 2. 等待全部完成
for ID in $IDLE1_ID $IDLE2_ID $IDLE3_ID \
          $TALK1_ID $TALK2_ID $TALK3_ID \
          $THINK1_ID $THINK2_ID $THINK3_ID \
          $LISTEN1_ID $LISTEN2_ID $LISTEN3_ID; do
  pixverse task wait $ID --json
done

# 3. 创建输出目录并下载，重命名为规范文件名
mkdir -p "output/$CHAR_NAME/videos"

# 逐一下载到临时目录后重命名，确保不混淆
for PAIR in \
  "$IDLE1_ID:idle1"     "$IDLE2_ID:idle2"     "$IDLE3_ID:idle3" \
  "$TALK1_ID:talk1"     "$TALK2_ID:talk2"     "$TALK3_ID:talk3" \
  "$THINK1_ID:think1"   "$THINK2_ID:think2"   "$THINK3_ID:think3" \
  "$LISTEN1_ID:listen1" "$LISTEN2_ID:listen2" "$LISTEN3_ID:listen3"; do
  VID_ID="${PAIR%%:*}"
  VID_NAME="${PAIR##*:}"
  TMP=$(mktemp -d)
  pixverse asset download "$VID_ID" --dest "$TMP" --json
  mv "$TMP"/*.mp4 "output/$CHAR_NAME/videos/${VID_NAME}.mp4"
  rm -rf "$TMP"
done

# 写入断点状态（视频全部下载完成）
cat > .pixverse_state.json << EOF
{
  "phase": "videos_done",
  "character_name": "$CHAR_NAME",
  "green_screen": $GREEN_SCREEN,
  "image_path": "output/$CHAR_NAME/image/character.png",
  "video_ids": {
    "idle1": "$IDLE1_ID", "idle2": "$IDLE2_ID", "idle3": "$IDLE3_ID",
    "talk1": "$TALK1_ID", "talk2": "$TALK2_ID", "talk3": "$TALK3_ID",
    "think1": "$THINK1_ID", "think2": "$THINK2_ID", "think3": "$THINK3_ID",
    "listen1": "$LISTEN1_ID", "listen2": "$LISTEN2_ID", "listen3": "$LISTEN3_ID"
  }
}
EOF
```

### 单视频重生机制

用户可对每个视频进行 **1次重生**（内部状态各项上限为1，粒度到 idle1/idle2 等）：

```
├── 用户满意全部 → 完成，提示可继续生成下一个角色
│
└── 用户指定某个视频重生（如 "idle2 不满意，动作太静了"）
     └── 检查对应计数是否已达上限（1次）
           ├── 未达上限
           │    └── 携带用户反馈修改提示词 → 重新生成该视频
           │         └── 下载并重命名替换，更新展示
           └── 已达上限
                └── 提示："该视频已无法继续重生，保留当前版本。"
```

**重生时的 CLI 命令**（替换提示词，不加 --no-wait，下载后覆盖对应编号文件）：
```bash
# 重新声明变量（bash 状态不跨 tool call 持久化）
CHAR_IMG="output/$CHAR_NAME/image/character.png"

NEW_RESULT=$(pixverse create transition \
  --images "$CHAR_IMG" "$CHAR_IMG" \
  --prompt "[根据用户反馈调整后的提示词]" \
  --quality 1080p --duration 3 \
  --json)
NEW_VIDEO_ID=$(echo "$NEW_RESULT" | jq -r '.video_id')

# 下载并重命名覆盖对应编号文件（如 idle2.mp4）
TMP=$(mktemp -d)
pixverse asset download "$NEW_VIDEO_ID" --dest "$TMP" --json
mv "$TMP"/*.mp4 "output/$CHAR_NAME/videos/[状态名][编号].mp4"
rm -rf "$TMP"
```

## 透明背景处理

所有视频验收完毕后，**仅当图像阶段使用了纯绿色（#00FF00）绿幕背景时**询问用户：
```
是否需要将图片和视频处理成透明背景版本？（适合叠加到不同场景使用）
```

若图像阶段选择了生成背景（内部状态 `绿幕背景: false`），跳过此询问，直接进入完成阶段。

```
├── 否 → 跳过，直接进入【完成后】
│
└── 是 → 自动对 12 个视频执行 ffmpeg 处理
```

### 处理说明

- **处理范围**：`image/` 下参考图 + `videos/` 下所有视频
- **抠绿幕**：使用 `colorkey` 滤镜去除绿色背景
- **Despill**：使用 `colorchannelmixer` 修正边缘绿色溢色
- **输出位置**：新建 `[image无背景版]/` 和 `[videos无背景版]/` 文件夹
- **输出格式**：图片 → PNG（含 Alpha）；视频 → WebM VP9（含 Alpha）

### 处理命令

```bash
# 创建无背景版文件夹
mkdir -p "output/$CHAR_NAME/image无背景版"
mkdir -p "output/$CHAR_NAME/videos无背景版"

# 1. 处理参考图
ffmpeg -i "output/$CHAR_NAME/image/character.png" \
  -vf "colorkey=color=0x00FF00:similarity=0.30:blend=0.08,\
       colorchannelmixer=rg=-0.15:bg=-0.15" \
  -pix_fmt rgba \
  "output/$CHAR_NAME/image无背景版/character无背景版.png"

# 2. 处理 12 个视频
for STATE in idle talk think listen; do
  for N in 1 2 3; do
    ffmpeg -i "output/$CHAR_NAME/videos/${STATE}${N}.mp4" \
      -vf "colorkey=color=0x00FF00:similarity=0.30:blend=0.08,\
           colorchannelmixer=rg=-0.15:bg=-0.15" \
      -c:v libvpx-vp9 \
      -pix_fmt yuva420p \
      "output/$CHAR_NAME/videos无背景版/${STATE}无背景版${N}.webm"
  done
done
```

```
       ├── talk无背景版1.webm / talk无背景版2.webm / talk无背景版3.webm
```
