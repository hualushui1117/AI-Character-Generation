## 阶段一：图像生成

### 入口识别

判断用户的初始信息，进入对应入口：

**入口 A — 用户上传了图片**
```
询问："你想直接使用这张图，还是以它为参考生成新图？"
├── 作为参考 → 进入入口 B（上传图 + 描述）
└── 直接使用
     └── 软提示："最佳格式为全身正面图 / 绿幕 / 9:16，你的图是否符合？"
           ├── 符合 → 跳过图像阶段，直接进入【角色命名】
           └── 不符合
                └── 执行图像命令（I2I），以原图为参考转换格式
                     └── 展示结果，进入【满意度循环】
```

**入口 B — 参考图 + 文字描述**
```
收集参考图路径 + 用户文字描述
└── 进入【引导追问机制】
     └── 执行图像命令（I2I），生成 2 张
          └── 进入【满意度循环】
```

**入口 C — 纯文字描述**
```
收集用户文字描述
└── 进入【引导追问机制】
     └── 执行图像命令（T2I），生成 2 张
          └── 进入【满意度循环】
```

---

### 引导追问机制

**5 个角色维度**（按入口 B/C 使用）：

| 维度 | 说明 | 示例 |
|------|------|------|
| 风格 | 整体画风 | 写实 / 动漫 / 3D / 像素 |
| 人设基础 | 性别、年龄感、种族 | 青年女性、精灵族 |
| 外貌特征 | 发色发型、眼睛、肤色 | 银白短发、红色眼睛 |
| 服装装扮 | 职业/风格/材质 | 黑色皮质战斗服、奇幻皮甲 |
| 气质表情 | 性格倾向与情绪基调 | 冷酷、活泼、神秘 |

**规则**：
- 用户未提供的维度由模型合理补全，不强制追问
- 每轮将所有缺失维度一次性批量问完（不分多轮追问）
- 最多追问 3 轮，超出后以现有信息直接生成
- 不向用户展示最终 prompt，直接执行生成

**示例**：
```
用户: "一个冷酷的女刺客"

已覆盖: 性别(女)、气质(冷酷)、职业暗示(刺客)
缺失:   风格、外貌、服装细节

Claude 回复:
还需要几个细节：
1. 画风偏写实还是动漫？
2. 发色和发型有想法吗？（如黑色短发 / 银白长发）
3. 服装风格？（现代战术装 / 奇幻皮甲 / 忍者风）
```

### 图像生成 CLI 命令

**T2I（纯文字，入口C）**：
```bash
pixverse create image \
  --prompt "全身正面图，绿幕背景，[用户角色描述]" \
  --model gemini-3.1-flash \
  --quality 1080p \
  --aspect-ratio 9:16 \
  --count 2 \
  --json
```

**I2I（参考图，入口A不符合/入口B）**：
```bash
pixverse create image \
  --prompt "全身正面图，绿幕背景，[用户角色描述]" \
  --image [参考图路径] \
  --model gemini-3.1-flash \
  --quality 1080p \
  --aspect-ratio 9:16 \
  --count 2 \
  --json
```

**解析输出**（count=2 时返回 image_ids 数组）：
```bash
IMAGE_IDS=$(echo "$RESULT" | jq -r '.image_ids[]')
# 或者直接从 image_url 获取（如果等待完成）
```

**下载图片**（先等待完成再下载）：

如果使用了 `--no-wait`，先等待：
```bash
for ID in $IMAGE_IDS; do
  pixverse task wait $ID --type image --json
done
```

下载（每张分别获取 info 以取得 image_url，展示给用户）：
```bash
for ID in $IMAGE_IDS; do
  pixverse asset info $ID --type image --json
done
```

用户选择后，下载选中的图片：
```bash
mkdir -p output/[角色名]/image
pixverse asset download [选中的image_id] --type image --dest output/[角色名]/image --json
```

### 满意度循环

每次展示 2 张图供用户选择，最多重生 3 次（内部状态图像重生次数）。

```
展示 2 张图（展示 asset info 中的 image_url）
 │
 ├── 用户选中一张 → 进入【角色命名】
 │
 ├── 局部不满意（有具体反馈）
 │    └── 定向修改对应维度 → 重新生成（图像重生次数 +1）
 │
 ├── 整体不满意
 │    └── 重新引导追问 → 重新生成（图像重生次数 +1）
 │
 └── 已达 3 次上限
      └── 提示："要不要重新描述角色，或者上传一张参考图？"
           ├── 是 → 重置重生次数，回到入口 B/C
           └── 否 → 从现有图中强制选一张继续
```

## 阶段二：视频生成

### 执行说明

- 系统**自动执行**，不向用户询问任何选择
- 参考图路径为阶段一下载的文件：`output/[角色名]/image/` 下的图片文件
- 先用 `--no-wait` 并行提交 4 个任务，再统一等待，最后下载

### 4 个状态的提示词

| 状态 | 提示词 |
|------|--------|
| idle | `角色静止站立，放松姿态，自然表情，镜头固定，保持竖屏构图` |
| talk | `角色正在说话，嘴部做出说话动作，热情表达，镜头固定，保持竖屏构图` |
| think | `角色做出思考的手势，若有所思，手指轻点下巴，镜头固定，保持竖屏构图` |
| listen | `角色侧耳倾听，认真聆听的表情，温和友善，镜头固定，保持竖屏构图` |

### 视频生成 CLI 命令

```bash
# 参考图路径（阶段一已下载的图片）
CHAR_IMG="output/[角色名]/image/[文件名]"

# 1. 并行提交 4 个任务（--no-wait 立即返回 video_id）
IDLE_RESULT=$(pixverse create transition \
  --images "$CHAR_IMG" "$CHAR_IMG" \
  --prompt "角色静止站立，放松姿态，自然表情，镜头固定，保持竖屏构图" \
  --quality 720p --duration 3 --no-wait --json)
IDLE_ID=$(echo "$IDLE_RESULT" | jq -r '.video_id')

TALK_RESULT=$(pixverse create transition \
  --images "$CHAR_IMG" "$CHAR_IMG" \
  --prompt "角色正在说话，嘴部做出说话动作，热情表达，镜头固定，保持竖屏构图" \
  --quality 720p --duration 3 --no-wait --json)
TALK_ID=$(echo "$TALK_RESULT" | jq -r '.video_id')

THINK_RESULT=$(pixverse create transition \
  --images "$CHAR_IMG" "$CHAR_IMG" \
  --prompt "角色做出思考的手势，若有所思，手指轻点下巴，镜头固定，保持竖屏构图" \
  --quality 720p --duration 3 --no-wait --json)
THINK_ID=$(echo "$THINK_RESULT" | jq -r '.video_id')

LISTEN_RESULT=$(pixverse create transition \
  --images "$CHAR_IMG" "$CHAR_IMG" \
  --prompt "角色侧耳倾听，认真聆听的表情，温和友善，镜头固定，保持竖屏构图" \
  --quality 720p --duration 3 --no-wait --json)
LISTEN_ID=$(echo "$LISTEN_RESULT" | jq -r '.video_id')

# 2. 等待全部完成
pixverse task wait $IDLE_ID   --json
pixverse task wait $TALK_ID   --json
pixverse task wait $THINK_ID  --json
pixverse task wait $LISTEN_ID --json

# 3. 创建输出目录并下载
mkdir -p "output/[角色名]/videos"
pixverse asset download $IDLE_ID   --dest "output/[角色名]/videos" --json
pixverse asset download $TALK_ID   --dest "output/[角色名]/videos" --json
pixverse asset download $THINK_ID  --dest "output/[角色名]/videos" --json
pixverse asset download $LISTEN_ID --dest "output/[角色名]/videos" --json
```

### 展示结果

视频生成完毕后，告知用户：

```
✅ [角色名] 的视频素材已生成完毕！

📁 output/[角色名]/
   ├── image/

   └── videos/

```

### 单视频重生机制

用户可对每个状态进行 **1次重生**（内部状态各项上限为1）：

```
├── 用户满意全部 → 完成，提示可继续生成下一个角色
│
└── 用户指定某个状态重生（如 "Talk 不满意，嘴部动作太小了"）
     └── 检查对应计数是否已达上限（1次）
           ├── 未达上限
           │    └── 可携带用户反馈修改提示词 → 重新生成该状态视频
           │         └── 下载替换，更新展示
           └── 已达上限
                └── 提示："该状态已无法继续重生，保留当前版本。"
```

**重生时的 CLI 命令**（与初次生成相同，替换提示词，不加 --no-wait）：
```bash
pixverse create transition \
  --images "$CHAR_IMG" "$CHAR_IMG" \
  --prompt "[根据用户反馈调整后的提示词]" \
  --quality 720p --duration 3 \
  --json

# 下载替换（覆盖原文件）
pixverse asset download [新video_id] --dest "output/[角色名]/videos" --json
```

## 错误处理

| 退出码 | 含义 | 处理方式 |
|--------|------|---------|
| 3 | 认证过期 | 提示用户运行 `pixverse auth login`，完成后重试 |
| 4 | Credits 不足 | 运行 `pixverse account info --json` 显示余额，提示用户充值后继续 |
| 5 | 生成失败 | 提示用户可能是内容审核问题，建议调整描述，询问是否重试 |
| 6 | 参数错误 | 检查命令参数后重试，告知用户具体问题 |
| 2 | 超时 | 提示用户网络可能较慢，询问是否重试 |

## 透明背景处理

所有视频验收完毕后，询问用户：
```
是否需要将视频处理成透明背景版本？（适合叠加到不同场景使用）
```

```
├── 否 → 跳过，直接进入【完成后】
│
└── 是 → 自动对 4 个视频执行 ffmpeg 处理
```

### 处理说明

- **处理范围**：`image/` 下所有图片 + `videos/` 下所有视频
- **抠绿幕**：使用 `colorkey` 滤镜去除绿色背景
- **Despill**：使用 `colorchannelmixer` 修正边缘绿色溢色
- **输出位置**：原文件夹内，文件名加 `_alpha` 后缀
- **输出格式**：图片 → PNG（含 Alpha）；视频 → WebM VP9（含 Alpha）

### 处理命令

```bash
# 1. 处理 image/ 下所有图片
for IMG in output/[角色名]/image/*.{png,jpg,jpeg}; do
  [ -f "$IMG" ] || continue
  FILENAME=$(basename "${IMG%.*}")
  ffmpeg -i "$IMG" \
    -vf "colorkey=color=0x00FF00:similarity=0.30:blend=0.08,\
         colorchannelmixer=rg=-0.15:bg=-0.15" \
    -pix_fmt rgba \
    "output/[角色名]/image/${FILENAME}_alpha.png"
done

# 2. 处理 videos/ 下所有视频
for VID in output/[角色名]/videos/*.mp4; do
  [ -f "$VID" ] || continue
  FILENAME=$(basename "${VID%.*}")
  ffmpeg -i "$VID" \
    -vf "colorkey=color=0x00FF00:similarity=0.30:blend=0.08,\
         colorchannelmixer=rg=-0.15:bg=-0.15" \
    -c:v libvpx-vp9 \
    -pix_fmt yuva420p \
    "output/[角色名]/videos/${FILENAME}_alpha.webm"
done
```

处理完成后告知用户：
```
✅ 透明背景版本已生成（原文件夹内，文件名含 _alpha）：

📁 output/[角色名]/
   ├── image/
   │   └── [原文件名]_alpha.png
   └── videos/
       ├── idle_alpha.webm
       ├── talk_alpha.webm
       ├── think_alpha.webm
       └── listen_alpha.webm
```
