## 这是什么

这是一个跑在 **Claude Code** 里的 AI 角色生成工作流。

你只需要用文字描述你想要的角色（比如"一个冷酷的银发女刺客"），或者上传一张参考图，工作流会自动：

1. 生成一张符合规范的 **全身正面角色参考图**（绿幕 / 9:16 竖屏）
2. 基于参考图，自动生成 **4 个状态共 12 条 3 秒循环动画视频**（每状态 3 个变体）：
   - **Idle（待机）**：角色静止站立，放松姿态
   - **Talk（说话）**：嘴部做出说话动作
   - **Think（思考）**：手指轻点下巴，若有所思
   - **Listen（聆听）**：侧耳倾听，专注表情

这些素材非常适合用于：游戏 NPC、虚拟主播、对话 UI、VN 视觉小说等场景。

## 环境准备

在开始之前，需要安装以下三个工具：

### 1. Node.js（版本 >= 20）

访问 [nodejs.org](https://nodejs.org) 下载安装。安装完成后验证：

```bash
node --version
# 应输出 v20.x.x 或更高
```

### 2. PixVerse CLI

PixVerse 是本工作流使用的 AI 生成服务。通过 npm 安装其命令行工具：

```bash
npm install -g pixverse
```

安装完成后，登录你的 PixVerse 账号（首次使用需要注册）：

```bash
pixverse auth login
# 会弹出一个设备授权流程，按提示在浏览器中完成登录
```

验证登录成功：

```bash
pixverse auth status --json
# 应显示你的账号信息
```

> **注意**：PixVerse 生成图像和视频需要消耗账号 Credits（积分）。请确保账号有足够余额，可通过 `pixverse account info --json` 查看余额。

### 3. ffmpeg（可选，仅透明背景处理时需要）

如果你需要将生成结果处理成**透明背景版本**（用于叠加不同场景），才需要安装 ffmpeg。

**macOS**（使用 Homebrew）：
```bash
brew install ffmpeg
```

**Windows**：
前往 [ffmpeg.org](https://ffmpeg.org/download.html) 下载，并将 `ffmpeg.exe` 所在目录添加到系统 PATH。

**Linux**：
```bash
sudo apt install ffmpeg   # Ubuntu/Debian
```

### 4. Claude Code

确保你在 **Claude Code** 客户端（桌面版 / VS Code 插件 / 命令行）中操作，并将工作目录切换到本项目根目录（即包含 `SKILL.md` 和 `docs/` 的目录）。

## 完整流程详解

### 阶段零：环境自检

工作流启动后，Claude 会自动检查：

- `pixverse` CLI 是否已安装
- 是否已登录 PixVerse 账号
- 当前工作目录是否是项目根目录

如果有任何一项不满足，Claude 会告知你需要完成的操作，完成后可以重新启动。

---

### 阶段一：生成角色参考图

#### 三种入口方式

根据你提供的初始信息，工作流会进入对应的入口：

---

**入口 A — 你上传了一张图片**

Claude 会询问你：

> "你想直接使用这张图，还是以它为参考生成新图？"

- **直接使用**：Claude 会软提示最佳格式要求（全身正面图 / 绿幕 / 9:16）
  - 如果你的图已符合 → 直接跳到角色命名，进入视频生成
  - 如果不符合 → 自动以原图为参考，生成一张符合格式的版本
- **以它为参考生成新图** → 进入入口 B

---

**入口 B — 参考图 + 文字描述**

你同时提供了参考图和文字描述，Claude 会根据描述找出视觉上存在合理分歧的维度，拟定两种不同诠释，然后生成 2 张候选图供你选择。

---

**入口 C — 纯文字描述**（最常用）

你只用文字描述角色，Claude 会通过**引导追问**帮你填充细节，然后针对描述中存在合理分歧的维度拟定两种诠释，分别各生成 1 张，共 2 张候选图。

---

#### 引导追问机制

Claude 会围绕以下 5 个维度询问缺失的信息：

| 维度 | 说明 | 示例 |
|------|------|------|
| 风格 | 整体画风 | 写实 / 动漫 / 3D / 像素 |
| 人设基础 | 性别、年龄感、种族 | 青年女性、精灵族 |
| 外貌特征 | 发色发型、眼睛、肤色 | 银白短发、红色眼睛 |
| 服装装扮 | 职业/风格/材质 | 黑色皮质战斗服、奇幻皮甲 |
| 气质表情 | 性格倾向与情绪基调 | 冷酷、活泼、神秘 |

**规则**：
- 你没提到的维度，Claude 会自动补全，不会强制追问
- 每轮把所有缺失维度**一次性批量问完**，不会反复来回
- 最多追问 3 轮，之后用已有信息直接生成
- Claude **不会展示最终 prompt**，直接开始生成

**示例对话：**

```
你：一个冷酷的女刺客

Claude：还需要几个细节：
  1. 画风偏写实还是动漫？
  2. 发色和发型有想法吗？（如黑色短发 / 银白长发）
  3. 服装风格？（现代战术装 / 奇幻皮甲 / 忍者风）
```

---

#### 满意度循环

每次生成会产出 **2 张候选图**（A/B 两种气质/风格诠释），你可以：

| 操作 | 说明 |
|------|------|
| 选中一张 | 进入角色命名步骤 |
| 局部不满意（有具体反馈） | Claude 定向修改对应维度，重新生成 |
| 整体不满意 | 重新追问，重新生成 |

> **重生上限：3 次**。用完后 Claude 会询问你是否重新描述或上传参考图。如果不想重来，可以从现有图中选一张继续。

---

#### 角色命名

选定参考图后，Claude 会提示：

> "请为这个角色取个名字～"

输入一个 2-20 字符的名字，比如 `冰川战士`、`月光精灵`。
这个名字会作为后续输出文件夹的名称。

---

### 阶段二：生成四个状态视频

这一阶段**完全自动执行**，你不需要做任何选择，等待即可。

Claude 会以阶段一确认的参考图为基础，同时提交 **12 个**视频生成任务（4 状态 × 3 变体）：

| 状态 | 说明 | 输出文件 | 时长 |
|------|------|----------|------|
| Idle（待机） | 角色静止站立，放松姿态，自然表情 | idle1/2/3.mp4 | 3 秒 |
| Talk（说话） | 角色正在说话，嘴部做出说话动作 | talk1/2/3.mp4 | 3 秒 |
| Think（思考） | 角色做出思考手势，手指轻点下巴 | think1/2/3.mp4 | 3 秒 |
| Listen（聆听） | 角色侧耳倾听，认真聆听的表情 | listen1/2/3.mp4 | 3 秒 |

每个状态生成 3 个变体，动作细节各有差异，方便你挑选最满意的版本。

> **技术说明**：视频使用 PixVerse 的 Transition 模式生成。首帧和尾帧都是同一张参考图（确保镜头固定、背景不变），通过不同的文字提示词驱动角色做出对应动作。每个状态的 3 条提示词来自 `prompt.md`，固定前缀保证镜头稳定和画面一致性。

生成完毕后，Claude 会展示输出文件路径，提示你查看视频。

---

#### 单视频重生机制

如果你对某个变体视频不满意，可以告诉 Claude：

> "talk2 不满意，嘴部动作太小了"

Claude 会根据你的反馈调整提示词，单独重新生成那一个视频。

**重生上限：每个视频最多 1 次**。超出后该视频无法继续重生，保留当前版本。

---

### 可选：透明背景处理

所有视频验收完毕后，Claude 会询问：

> "是否需要将图片和视频处理成透明背景版本？（适合叠加到不同场景使用）"

选择 **是** 后，Claude 会用 ffmpeg 自动处理图片和视频，去除绿色背景。

| 原文件 | 处理后 |
|--------|--------|
| `character.png` | `image无背景版/character无背景版.png`（透明背景 PNG） |
| `idle1/2/3.mp4` | `videos无背景版/idle无背景版1/2/3.webm`（透明背景 WebM VP9） |
| `talk1/2/3.mp4` | `videos无背景版/talk无背景版1/2/3.webm` |
| `think1/2/3.mp4` | `videos无背景版/think无背景版1/2/3.webm` |
| `listen1/2/3.mp4` | `videos无背景版/listen无背景版1/2/3.webm` |

> **技术说明**：使用 `colorkey` 滤镜抠除绿色背景，并用 `colorchannelmixer` 修正边缘绿色溢色（Despill）。图片输出为 RGBA PNG，视频输出为 WebM（yuva420p）格式。

## 输出文件结构

生成完成后，所有文件会保存在项目根目录的 `output/` 文件夹中：

```
output/
└── [角色名]/                        # 例：冰川战士
    ├── image/
    │   └── character.png            # 原始参考图（绿幕，9:16）
    ├── image无背景版/               # 透明背景版（可选）
    │   └── character无背景版.png
    ├── videos/
    │   ├── idle1.mp4 / idle2.mp4 / idle3.mp4
    │   ├── talk1.mp4 / talk2.mp4 / talk3.mp4
    │   ├── think1.mp4 / think2.mp4 / think3.mp4
    │   └── listen1.mp4 / listen2.mp4 / listen3.mp4
    └── videos无背景版/              # 透明背景版（可选）
        ├── idle无背景版1.webm / idle无背景版2.webm / idle无背景版3.webm
        ├── talk无背景版1.webm / talk无背景版2.webm / talk无背景版3.webm
        ├── think无背景版1.webm / think无背景版2.webm / think无背景版3.webm
        └── listen无背景版1.webm / listen无背景版2.webm / listen无背景版3.webm
```

生成多个角色时，每个角色独立存放：

```
output/
├── 冰川战士/
├── 月光精灵/
└── 暗夜探手/
```

## 错误处理说明

| 错误码 | 含义 | 解决方法 |
|--------|------|---------|
| 2 | 超时 | 网络较慢，Claude 会询问是否重试 |
| 3 | 认证过期 | 重新运行 `pixverse auth login` 登录后重试 |
| 4 | Credits 不足 | 运行 `pixverse account info --json` 查看余额，充值后继续 |
| 5 | 生成失败 | 可能触发了内容审核，尝试调整角色描述（避免敏感词）后重试 |
| 6 | 参数错误 | Claude 会自动检查参数并修正后重试 |

## 常见问题

**Q：一次生成需要消耗多少 Credits？**

大约消耗：图像生成（2 张）+ 视频生成（12 条）。具体消耗量以 PixVerse 官网当前定价为准。

---

**Q：生成的视频为什么镜头固定、背景不动？**

这是有意为之的设计。使用首尾帧相同的参考图，可以确保镜头不动、背景固定，只有角色本身做出动作。这样的视频可以直接叠加在任意背景上使用，是游戏/应用 UI 素材的标准做法。

---

**Q：透明背景处理需要 ffmpeg，但我不想安装怎么办？**

透明背景处理是可选步骤，跳过没有任何影响。原始的 MP4 视频（绿幕）同样可以直接使用，只是无法叠加在非绿色背景上。

---

**Q：我可以批量生成多个角色吗？**

可以。单个角色全部完成后，Claude 会询问是否继续生成下一个角色。每次从头描述即可，输出文件互相独立，不会覆盖。

---

**Q：可以用真人照片作为参考图吗？**

可以上传照片作为参考，但最终生成结果会受到 PixVerse 内容审核限制。建议使用插画/动漫风格的参考图，成功率更高。

---

**Q：生成的视频分辨率是多少？**

- 参考图：1080p，9:16 竖屏
- 视频：1080p，9:16 竖屏，3 秒时长

---

全部完成后，Claude 会提示：

```
这个角色的素材已全部完成 🎉
要继续生成下一个角色吗？（从头开始 /角色生成）
```
