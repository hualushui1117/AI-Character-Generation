# Workflow — 图像生成阶段

## 概述

使用 **Pixverse 图片生成工具**，用户通过文字描述或上传图片，生成符合固定格式规范的角色参考图，作为后续视频生成的素材基础。

---

## 生成工具

- **工具**：Pixverse `create image` 命令
- **模型**：`gemini-3.1-flash`（Google Gemini 3.1）
- **质量**：720p
- **分辨率比例**：9:16（竖屏）

---

## 固定 Prompt 规范

所有生成图片必须遵循以下固定基础 prompt：

```
全身正面图，尺寸9:16
```

其中 **「绿幕背景」为条件项**，根据"背景确认"步骤的选择决定是否加入：

```
├── 用户选择无背景（默认）→ 全身正面图，纯绿色（#00FF00）绿幕背景，尺寸9:16，[角色描述]
└── 用户选择生成背景    → 全身正面图，尺寸9:16，[角色描述]，[背景描述（可选）]
```

**最终 prompt 结构（无背景）：**
```
全身正面图，纯绿色（#00FF00）绿幕背景，尺寸9:16，[用户角色描述]
```

**最终 prompt 结构（有背景）：**
```
全身正面图，尺寸9:16，[用户角色描述]，[背景描述（可选）]
```

**示例（无背景）：**
```
全身正面图，纯绿色（#00FF00）绿幕背景，尺寸9:16，冷酷女刺客，银白短发，红色眼睛，黑色皮质战斗服，动漫风格
```

## 入口流程

### 入口 A — 用户有现成图片

```
用户上传图片
 └── Claude 询问："你想直接使用这张图，还是以它为参考生成新图？"
      │
      ├── 作为参考 ──────────────────────────────→ 跳转入口 B
      │
      └── 直接使用
           └── 软提示："最佳格式为全身正面图 / 绿底 / 9:16，你的图是否符合？"
                │
                ├── 符合 ──────────────────────────→ 进入视频生成阶段
                │
                └── 不符合
                     └── 以原图为参考，自动生成符合固定格式的版本
                          └── 展示生成结果给用户确认
                               ├── 满意 ───────────→ 进入视频生成阶段
                               └── 不满意 ─────────→ 跳转入口 B
```

---

### 入口 B — 参考图 + 文字描述

```
用户上传参考图 + 提供文字描述
 └── Claude 引导追问缺失的角色维度（见"引导追问机制"）
      └── 背景确认：「需要为角色生成背景吗？」
           ├── 是 → 移除纯绿色（#00FF00）绿幕背景，软提示无法透明背景处理 → 固定 prompt（无绿幕） + 用户描述 → 生成2张图
           └── 否 → 固定 prompt（含绿幕） + 用户描述 → 生成2张图
                └── 进入满意度循环
```

---

### 入口 C — 纯文字描述

```
用户提供文字描述
 └── Claude 引导追问缺失的角色维度（见"引导追问机制"）
      └── 背景确认：「需要为角色生成背景吗？」
           ├── 是 → 移除纯绿色（#00FF00）绿幕背景，软提示无法透明背景处理 → 固定 prompt（无绿幕） + 用户描述 → 生成2张图
           └── 否 → 固定 prompt（含绿幕） + 用户描述 → 生成2张图
                └── 进入满意度循环
```

## 引导追问机制

### 角色维度表

| 维度 | 说明 | 示例 |
|------|------|------|
| 风格 | 整体画风 | 写实 / 动漫 / 3D / 像素 |
| 人设基础 | 性别、年龄感、种族 | 青年女性、精灵族 |
| 外貌特征 | 发色发型、眼睛、肤色 | 银白短发、红色眼睛 |
| 服装装扮 | 职业/风格/材质 | 黑色皮质战斗服、奇幻皮甲 |
| 气质表情 | 性格倾向与情绪基调 | 冷酷、活泼、神秘 |

### 追问规则

- **模式：宽松** — 用户未提及的维度由模型合理补全，不强制追问
- **上限：3轮** — 最多追问3轮，超出后以现有信息生成
- **方式：批量** — 每轮将所有缺失维度一次性问完，避免多次来回
- **不展示最终 prompt** — 直接生成，减少用户认知负担

**示例：**
```
用户: "一个冷酷的女刺客"

已覆盖: 性别(女)、气质(冷酷)、职业暗示(刺客)
缺失:   风格、外貌、服装细节

Claude: 还需要几个细节：
  1. 画风偏写实还是动漫？
  2. 发色和发型有想法吗？（如黑色短发 / 银白长发）
  3. 服装风格？（现代战术装 / 奇幻皮甲 / 忍者风）
```

## 满意度循环

每次生成产出 **2张图** 供用户选择，最多重生 **3次**。

```
生成2张图
 │
 ├── 用户选中一张
 │    └── [新增] Claude 请求用户为角色取名
 │         └── 用户输入角色名（E.g., "冰川战士"、"月光精灵"）
 │              └── 确认角色名
 │                   └── 进入视频生成阶段（角色名作为后续文件夹命名）
 │
 ├── 局部不满意（有具体反馈）
 │    └── 主动询问："还有什么想修改或增加的关键词吗？"
 │         └── 整合用户反馈 → 定向修改对应维度 → 重新生成2张（计入重生次数）
 │
 ├── 整体不满意
 │    └── 主动询问："有什么想修改或增加的关键词吗？"
 │         └── 整合用户反馈 → 重新进入引导追问 → 重新生成2张（计入重生次数）
 │
 └── 3次用尽仍不满意
      └── 提示用户："要不要重新描述角色，或者上传一张参考图？"
```

## 完整流程图

```
开始
 │
 ├── [入口A] 用户有现成图
 │     └── 询问意图
 │           ├── 作为参考 ──────────────────→ [入口B]
 │           └── 直接使用
 │                 └── 软提示格式要求
 │                       ├── 符合 ──────────→ 角色命名 → 视频生成阶段
 │                       └── 不符合
 │                             └── 自动转换为固定格式 → 用户确认
 │                                   ├── 满意 ──────→ 角色命名 → 视频生成阶段
 │                                   └── 不满意 ────→ [入口B]
 │
 ├── [入口B] 参考图 + 文字描述
 │     └── 引导追问 → 背景确认 → 生成2张 → 满意度循环 → 角色命名 → 视频生成阶段
 │
 └── [入口C] 纯文字描述
       └── 引导追问 → 背景确认 → 生成2张 → 满意度循环 → 角色命名 → 视频生成阶段
```

## 角色命名规范

当用户确定参考图后，系统提示用户输入角色名：

- **提示文本**：`请为这个角色取个名字～`
- **输入格式**：自由文本，2-20 字符
- **示例**：冰川战士、月光精灵、暗夜探手、浮云仙子
- **用途**：后续作为视频文件夹的名称，便于管理和识别
